# artx クックブック — 良い artx の作り方

artx を使って良いデザインを作るための**知恵（ベストプラクティス）**。
artx の XML 形式・タグ・属性の文法は `artx-format.md`（リファレンス）を引く。
ここは「文法を知っている前提で、どう組み立てれば良いデザインになるか」を扱う。

## テンプレートの選び方 — 一覧の keywords で絞る

- 環境のテンプレ一覧（システムプロンプト「## 環境」）は各 .artx を**絶対パス＋
  keywords** で示す。例: `- /path/xmas_frame.artx — [冬, クリスマス, 枠]`。
- まず **keywords で候補を絞る**。keywords は **preview（見た目）では分からない意図**
  ——季節・行事（クリスマス・母の日 等）・用途・対象——を表す。依頼の文脈（「クリス
  マス用の」「枠だけ」等）に合うタグのテンプレを選ぶ。逆に「空・水・枠」のような
  見れば分かる題材はタグに無くてよい（render_view で見て判断する側）。
- 詳細が要るときは `getArtxInfo(path)`（`{ width, height, version, fields,
  imageFields, keywords }` を返す）、中身を読むなら `read_artx(path)`。**パスは一覧の
  絶対パスをそのまま渡せる**（place / getArtxInfo / read_artx すべて絶対パス可）。
- 合うものが無ければ下記「使うか・作るか・改変するか」へ。

## テンプレートを使うか、その場で作るか、改変するか

- 依頼に合うテンプレートが無いときは、**無関係な既存テンプレート（card 等）を
  流用しない**。`placeArtxSource()` で依頼に合った**最小の artx をその場で生成**する
  （例: 「『〇〇』という文字列を描いて」→ card を使い回さず、テキスト 1 個だけの
  artx を作る）。
- 既存テンプレートをそのまま使えるなら `place()` で配置する。
- **既存テンプレートを下敷きにカスタマイズしたいとき** — 色・フォント・要素・
  レイアウトを変えたい — は、`read_artx` ツールでそのテンプレの XML を読み、
  改変した XML を `placeArtxSource()` で配置する（テンプレは固定資産ではなく
  **出発点**。読んで → 好みに変えて → 置く）。

## テキストの種類: ポイント vs エリア

- 特に理由がなければ **ポイントテキスト**（`<text>` の `type` を省略）。単純な
  文字列・見出し・ラベルはこれ。人間が普通にテキストを置くときの既定。
- 決まった枠内に収めて ArtTracer の**溢れ自動調整（autoFit）で自動的に詰めたい
  ときだけ** エリアテキスト（`type="area"` + 境界 `<path>`）を使う。
- ポイントテキストは溢れの概念がなく autoFit の対象外（だから「枠に収める」目的が
  ないならポイントで十分）。

## 重ね順（z-order）の組み立て — 3階層と層のグループ化

レイアウトの重ね順には、上手な作り手が必ず守る**3階層**がある（背面→前面）。
1 アートボード全体でも、artx のような小さな単位でも**同じ**（スケール不変）。

| 階層 | z 位置 | 中身 |
|------|--------|------|
| **3. テキスト** | 最前面 | 見出し・本文・価格・ロゴ文字（必ず読めるよう一番上） |
| **2. 中間デザイン** | 中間 | 装飾・画像・帯・図形・アクセント |
| **1. 枠** | 最背面 | 背景・地色・コンテナ・罫線（中身を載せる「器」） |

これを破ると「文字が図形に隠れる」「背景が前に出て読めない」など機能的に破綻する。
artx の記述順は **先＝前面 / 後＝背面**（`artx-format.md` §1.5）なので、生成時は
**前景（テキスト）を先に、背景（枠）を最後に**書く。

### 編集させたい層は `<group>` でまとめる

各層、特に**枠は複数要素（背景矩形＋地色帯＋罫線…）で構成されることが多い**。
これらを 1 つの `<group>` に包むと:

- その層が **1 つのまとまり（1 ハンドル）**になり、配置後に id で**層ごと丸ごと**
  移動・拡縮・差し替えできる（`stretchById` で枠を領域にフィット、など）
- 「どこからどこまでが枠か」の**境界が構造で明示**される（推測不要）

逆にグループ化しない場合、後から枠を指すときは **最背面の非テキスト要素 1 つ**しか
枠とみなせない。複数要素を 1 つの枠として扱いたいなら、**必ずグループ化する**。

## 良いデザインの土台

- レイアウトは `めざすデザインについて.md` のデザイン4原則
  （**近接・整列・反復・対比**）を踏まえる。関連要素は近づけ、配置基準を揃え、
  色やフォントを反復させ、強調部にメリハリをつける。
- 余白の活用・視線誘導（左上→右下）・ブロック化も同 doc を参照。
