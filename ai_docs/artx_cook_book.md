# artx クックブック — 良い artx の作り方

artx を使って良いデザインを作るための**知恵（ベストプラクティス）**。
artx の XML 形式・タグ・属性の文法は `artx-format.md`（リファレンス）を引く。
ここは「文法を知っている前提で、どう組み立てれば良いデザインになるか」を扱う。

## テンプレートを使うか、その場で作るか、改変するか

- 依頼に合うテンプレートが無いときは、**無関係な既存テンプレート（card 等）を
  流用しない**。`placeArtxSource()` で依頼に合った**最小の artx をその場で生成**する
  （例: 「『〇〇』という文字列を描いて」→ card を使い回さず、テキスト 1 個だけの
  artx を作る）。
- 既存テンプレートをそのまま使えるなら `place()` で配置する。
- **既存テンプレートを下敷きにカスタマイズしたいとき** — 色・フォント・要素・
  レイアウトを変えたい — は、`read_artx` ツールでそのテンプレの XML を読み、
  改変した XML を `placeArtxSource()` で配置する（テンプレは固定資産ではなく
  **出発点**。読んで → 好みに変えて → 置く）。

## テキストの種類: ポイント vs エリア

- 特に理由がなければ **ポイントテキスト**（`<text>` の `type` を省略）。単純な
  文字列・見出し・ラベルはこれ。人間が普通にテキストを置くときの既定。
- 決まった枠内に収めて ArtTracer の**溢れ自動調整（autoFit）で自動的に詰めたい
  ときだけ** エリアテキスト（`type="area"` + 境界 `<path>`）を使う。
- ポイントテキストは溢れの概念がなく autoFit の対象外（だから「枠に収める」目的が
  ないならポイントで十分）。

## 良いデザインの土台

- レイアウトは `めざすデザインについて.md` のデザイン4原則
  （**近接・整列・反復・対比**）を踏まえる。関連要素は近づけ、配置基準を揃え、
  色やフォントを反復させ、強調部にメリハリをつける。
- 余白の活用・視線誘導（左上→右下）・ブロック化も同 doc を参照。
